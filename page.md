## 1. 全体構成

### 1.1 ページ階層構造

```
/                           # ホーム
├── /aladdin                # 要件定義AI（アラジン）
│   ├── /aladdin/start      # ヒアリング開始
│   ├── /aladdin/result     # 仕様書生成結果
│   └── /aladdin/history    # 過去の仕様書一覧
│
├── /market                 # 相場検索
│   ├── /market/search      # 相場検索・結果
│   └── /market/report      # 相場レポート詳細
│
├── /vendors                # ベンダー一覧
│   ├── /vendors/:id        # ベンダー詳細
│   └── /vendors/compare    # ベンダー比較
│
├── /cases                  # 導入事例・パターン
│   ├── /cases/:industry    # 業界別事例
│   └── /cases/:id          # 事例詳細
│
├── /dashboard              # ダッシュボード（ログイン後）
│   ├── /dashboard/projects # プロジェクト管理
│   ├── /dashboard/specs    # 仕様書管理
│   └── /dashboard/settings # 設定
│
├── /vendor-portal          # ベンダー向けポータル
│   ├── /vendor-portal/profile    # プロファイル編集
│   ├── /vendor-portal/leads      # 案件一覧
│   └── /vendor-portal/analytics  # 分析・レポート
│
├── /auth                   # 認証
│   ├── /auth/login         # ログイン
│   ├── /auth/register      # 新規登録
│   └── /auth/onboarding    # オンボーディング
│
└── /pages                  # 静的ページ
    ├── /about              # 会社概要
    ├── /how-it-works       # 使い方
    ├── /pricing            # 料金
    ├── /contact            # お問い合わせ
    ├── /terms              # 利用規約
    └── /privacy            # プライバシーポリシー
```

## 3. ホームページ

### 3.1 ページ目的

- プラットフォームの価値を30秒で理解させる
- 主要機能への誘導
- 信頼性の担保（実績数・登録社数）

### 3.2 セクション構成

#### セクション1: ヒーロー
**配置**: ファーストビュー全画面

- **ヘッドライン**: 「AI開発、もう迷わない」
- **サブヘッドライン**: 「仕様書を自動作成、相場がわかる、最適なベンダーが見つかる」
- **主要CTA**:
  - 「無料で要件定義を始める」（プライマリ）
  - 「相場を調べる」（セカンダリ）
- **背景**: 抽象的なデータビジュアライゼーション or グラデーション

#### セクション2: 3つの価値提案
**配置**: ヒーロー直下

3カラム構成：

| カード | アイコン | タイトル | 説明 |
|-------|---------|---------|------|
| 1 | 💬 | 対話するだけで仕様書完成 | AIとの対話で要件を整理。専門知識不要で標準フォーマットの仕様書を自動生成 |
| 2 | 📊 | 相場が一目でわかる | 業界×ユースケース別の価格レンジを可視化。見積もりの妥当性を即座に判断 |
| 3 | 🏢 | 最適なベンダーを発見 | 強み・実績・評価で比較。あなたの案件にマッチする開発会社を提案 |

#### セクション3: 利用の流れ
**配置**: 3ステップの横並び

```
STEP 1           STEP 2           STEP 3
[アイコン]       [アイコン]       [アイコン]
要件を整理       相場を確認       ベンダー選定
────────────→  ────────────→  ────────────→

AIと対話して      類似案件の       評価・実績で
仕様書を作成      価格帯を確認     最適な会社を選ぶ
```

#### セクション4: 数字で見る実績
**配置**: 中央揃え、大きな数字

| 指標 | 数値 | 補足 |
|-----|------|------|
| 登録ベンダー数 | XX社 | AI開発専門企業 |
| 仕様書生成数 | X,XXX件 | 累計 |
| 相場データ | XXX件 | 業界×ユースケース |
| 平均マッチング日数 | X日 | 問い合わせから |

#### セクション5: 相場クイック検索
**配置**: カード型の検索UI

- **見出し**: 「今すぐ相場を調べる」
- **入力要素**:
  - 業界選択（ドロップダウン）
  - ユースケース選択（ドロップダウン）
  - 検索ボタン
- **サンプル結果**: 「製造業×外観検査：300〜600万円」のような表示

#### セクション6: 導入事例カルーセル
**配置**: 横スクロールカード

各カードに含む要素：
- 業界タグ
- 導入内容（1行）
- 価格帯
- 期間
- 「詳しく見る」リンク

#### セクション7: ベンダー紹介
**配置**: ロゴ一覧 + 注目ベンダーカード3枚

- **見出し**: 「信頼できるAI開発会社が集結」
- **ロゴ一覧**: パートナー企業のロゴ（許可済みのもの）
- **注目ベンダーカード**:
  - 会社名
  - 得意分野タグ
  - 評価スコア
  - 「詳細を見る」

#### セクション8: 発注者/ベンダー向けCTA
**配置**: 2カラム

| 発注者向け | ベンダー向け |
|-----------|-------------|
| 「AI開発を検討中の方へ」 | 「AI開発会社の方へ」 |
| 無料で仕様書作成 | 案件獲得チャネルを増やす |
| 相場感を把握 | 質の高いリードを獲得 |
| [要件定義を始める] | [パートナー登録] |

#### セクション9: FAQ
**配置**: アコーディオン形式

想定FAQ：
- 「利用料金はかかりますか？」
- 「どんな案件が対象ですか？」
- 「相場データはどこから？」
- 「ベンダー登録の条件は？」

#### セクション10: CTA（最終）
**配置**: フルワイドバナー

- **コピー**: 「AI開発の第一歩を、今日から」
- **CTA**: 「無料で始める」

## 4. 要件定義AIページ（アラジン）

### 4.1 ページ目的

- 対話形式で要件をヒアリング
- 標準化された仕様書を自動生成
- 相場レンジ・推奨ベンダーも同時に提示

### 4.2 ページ構成

#### 4.2.1 ランディング画面 `/aladdin`

- **ヘッドライン**: 「対話するだけで、AI開発の仕様書が完成」
- **サブコピー**: 「10〜15の質問に答えるだけ。専門知識は不要です」
- **CTA**: 「無料で始める」
- **期待設定**: 「所要時間：約5〜10分」
- **サンプル表示**: 生成される仕様書のプレビュー画像

#### 4.2.2 ヒアリング画面 `/aladdin/start`

**レイアウト**: チャット形式 or ステップ形式

##### チャット形式の場合

```
┌─────────────────────────────────────┐
│ [進捗バー: 5/15]                     │
├─────────────────────────────────────┤
│                                     │
│  🤖 「どの業界のお仕事ですか？」      │
│                                     │
│     [製造業] [小売・流通] [金融]     │
│     [医療・ヘルスケア] [不動産]      │
│     [その他...]                     │
│                                     │
├─────────────────────────────────────┤
│ [戻る]                    [スキップ] │
└─────────────────────────────────────┘
```

##### 回答形式

| タイプ | 使用場面 |
|-------|---------|
| 単一選択 | 業界、規模など |
| 複数選択 | 課題、技術要件など |
| はい/いいえ | 既存システム有無など |
| テキスト入力 | 詳細説明、補足 |
| スライダー | 予算レンジ、期間など |

##### 質問フロー例

1. 業界は？
2. 解決したい課題は？（複数選択可）
3. 現在の業務フローは？
4. 既存データはある？
5. 既存システムとの連携は必要？
6. 社内にIT担当者はいる？
7. 想定予算は？
8. 希望納期は？
...

#### 4.2.3 結果画面 `/aladdin/result`

**構成**:
